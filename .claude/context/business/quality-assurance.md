# 품질 보증

## 목적

코드 변경이 main 브랜치에 병합되기 전에 자동화된 통합 테스트를 실행하여, 결제/주문 등 핵심 비즈니스 로직의 정상 동작을 검증한다. 실제 데이터베이스 환경에서 서비스 레이어를 테스트하여 배포 안정성을 확보한다.

## 핵심 기능

| 기능 | 설명 | 사용자 관점 |
|------|------|------------|
| PR 자동 테스트 | PR 생성/업데이트 시 통합 테스트 자동 실행 | 개발자가 코드 변경의 안전성을 즉시 확인 |
| 실환경 유사 테스트 | Docker로 PostgreSQL 등 실제 인프라 구동 | 배포 전 실환경과 동일한 조건에서 검증 |
| 커버리지 수집 | 테스트 커버리지 리포트 생성 및 보존 | 팀이 테스트 범위를 지속적으로 모니터링 |

## 사용자 흐름

### 개발자 PR 제출

1. 개발자가 feature 브랜치에서 코드 작성 후 main 대상 PR 생성
2. GitHub Actions가 자동으로 통합 테스트 워크플로우 실행
3. Docker Compose로 테스트용 데이터베이스 구동
4. Service Layer 통합 테스트 실행 (실제 DB 연동)
5. 테스트 결과가 PR에 표시 -> 통과 시 병합 가능

### 코드 업데이트

1. PR에 새 커밋 push 시 테스트 자동 재실행
2. 이전 테스트 실패 원인을 수정했는지 즉시 확인 가능

## 비즈니스 규칙

- main 브랜치 대상 PR에서만 통합 테스트를 실행하여 CI 리소스를 효율적으로 사용한다
- 테스트 실패 시에도 인프라 정리가 보장되어 리소스 누수를 방지한다
