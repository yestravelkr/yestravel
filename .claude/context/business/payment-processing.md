# 결제 처리

## 목적

고객이 상품을 선택하고 결제를 완료하면, 임시 주문을 정식 주문으로 확정하고 PG사(포트원)를 통해 결제를 승인한다. 관리자가 주문을 취소할 경우 PG사를 통해 전액 또는 부분 환불을 처리한다.

## 핵심 기능

| 기능 | 설명 | 사용자 관점 |
|------|------|------------|
| 결제 완료 처리 | 임시 주문을 정식 주문으로 변환하고 PG 승인 | 고객이 결제 버튼 클릭 후 주문이 확정됨 |
| 주문 타입별 처리 | 호텔/배송/E-티켓에 따라 적절한 주문 생성 | 상품 유형에 관계없이 동일한 결제 경험 |
| 결제 취소 | PG사를 통한 전액/부분 취소 | 관리자가 주문 취소 시 자동 환불 처리 |
| 결제 이력 관리 | Payment 엔티티로 결제 상세 정보 보존 | 고객/관리자가 결제 내역 확인 가능 |

## 사용자 흐름

### 결제 완료

1. 고객이 상품 선택 후 결제 진행 -> 임시 주문(TmpOrder) 생성
2. PG사(포트원) 결제창에서 결제 수행
3. 결제 완료 시 시스템이 임시 주문을 정식 주문으로 변환
4. PG사에 결제 승인 요청 -> 승인 완료 -> 결제 정보 저장
5. 고객에게 주문 완료 안내

### 결제 취소 (관리자)

1. 관리자가 백오피스에서 주문 취소 요청 (환불 금액 직접 지정)
2. 시스템이 주문 상태 및 결제 정보 검증
3. PG사(포트원)에 환불 요청 전송 (관리자가 지정한 금액)
4. 주문 상태를 CANCELLED로 변경, 결제 잔액 업데이트
5. 고객에게 환불 처리

## 비즈니스 규칙

- 결제 승인이 완료된 후 결제 정보 저장에 실패하더라도, 이미 승인된 결제는 취소하지 않는다 (데이터 정합성보다 결제 안정성 우선)
- 주문번호(orderNumber)는 임시 주문 ID를 기반으로 인코딩되어 PG사와 통신에 사용된다
- 이미 결제된 요청이 중복 도달하면 에러 없이 정상 처리한다 (멱등성)
- 주문 취소 시 관리자가 환불 금액(refundAmount)을 직접 지정한다 (전액 또는 부분 환불)
- 이미 취소된 주문은 중복 취소할 수 없다
