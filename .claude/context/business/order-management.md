# 주문 관리

## 목적

백오피스 관리자가 고객 주문의 전체 라이프사이클을 관리할 수 있도록 지원. 주문 확인, 예약 확정, 주문 취소, 취소 요청 승인/거절 등의 기능을 제공한다.

## 핵심 기능

| 기능 | 설명 | 사용자 관점 |
|------|------|------------|
| 판매자 직접 취소 | 수수료와 사유를 입력하여 관리자가 주문을 직접 취소 | 재고 부족, 가격 오류 등 판매자 사유로 즉시 환불 처리 |
| 취소 수수료 설정 | 취소 시 수수료를 지정하고 환불 금액 자동 계산 | 상품금액에서 수수료를 차감한 금액이 고객에게 환불 |
| 취소 사유 관리 | 사전 정의된 사유 선택 또는 직접 입력 | 취소 이력 추적 및 통계 활용 |
| 취소 요청 승인/거절 | 고객이 요청한 취소를 관리자가 승인하거나 거절 | 고객 취소 요청에 대한 판매자 의사결정 |
| 주문 상태 변경 | 결제완료 → 예약대기 → 예약확정 순서로 상태 진행 | 주문 처리 진행 상황 관리 |
| 상태 되돌리기 | 잘못 변경한 상태를 이전 단계로 되돌림 | 관리자 실수 복구 |

## 사용자 흐름

1. 관리자가 호텔 주문 상세 페이지에서 "주문취소" 클릭 → 취소 모달에서 수수료/사유 입력 → 최종 확인 → 즉시 환불 처리
2. 고객이 취소 요청 → 관리자가 취소요청 상세 확인 → 승인(수수료 설정 후 환불) 또는 거절(이전 상태 복원)
3. 관리자가 주문 확인 → 예약대기 → 예약확정 순서로 상태를 진행하며, 필요 시 이전 단계로 되돌림

## 주문 상태 흐름

```
PAID → PENDING_RESERVATION → RESERVATION_CONFIRMED → COMPLETED
         ↑ (되돌리기)              ↑ (되돌리기)

어느 단계에서든 → CANCELLED (판매자 직접 취소)
CANCEL_REQUESTED → CANCELLED (승인) 또는 이전 상태 (거절)
```
