# 주문 상태 관리

## 목적

백오피스 관리자가 주문의 상태를 조회하고 관리할 수 있도록 한다. 상태 전이 규칙에 따른 안전한 상태 변경, 실수 시 되돌리기, 어드민 직접 취소/환불 기능을 제공한다.

## 핵심 기능

| 기능 | 설명 | 사용자 관점 |
|------|------|------------|
| 주문 목록 조회 | 필터/검색/페이지네이션으로 주문 검색 | 관리자가 원하는 주문을 빠르게 찾음 |
| 상태별 카운트 | 각 상태의 주문 수를 표시 | 대시보드에서 현황 파악 |
| 상태 변경 | 허용된 전이 규칙에 따라 상태 변경 | 주문 처리 단계 진행 (예: 배송준비→배송중) |
| 상태 되돌리기 | 호텔 주문의 상태를 이전 단계로 복원 | 실수로 상태를 잘못 변경한 경우 복구 |
| 어드민 직접 취소 | PG사 환불 API 호출 + 주문 취소 | 고객 요청 없이 관리자가 직접 취소/환불 처리 |
| 엑셀 내보내기 | 주문 데이터를 엑셀로 다운로드 | 정산/보고용 데이터 추출 |

## 사용자 흐름

1. 관리자가 백오피스 주문 목록 페이지에 진입 → 상태별 카운트와 주문 목록 표시
2. 필터/검색으로 원하는 주문 찾기 → 주문 상세 페이지로 이동
3. 주문 상태 변경 → 허용된 전이만 가능, 규칙 위반 시 에러
4. 상태를 잘못 변경한 경우 → 되돌리기 버튼으로 이전 상태 복원 (호텔 주문만)
5. 주문 취소가 필요한 경우 → 어드민 직접 취소로 PG 환불 + 상태 변경

## 비즈니스 규칙

- 주문 상태는 12단계 (PENDING → RETURNED)
- 상태 전이는 `canTransition` 규칙으로 제한 (아무 상태로든 변경 불가)
- 상태 되돌리기는 호텔(HOTEL) 상품에만 지원
- 표시 상태(displayStatus)는 claim 정보를 합산하여 CANCEL_REQUESTED, RETURN_REQUESTED를 추가 표시
