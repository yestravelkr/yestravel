# 추가결제 Context

## 사용자 요청 원문

> 결제해서 주문이 생성됐을 때 해당 주문에 추가결제를 할 수 있는 기능을 붙일거야
>
> order가 있을 경우에만 가능하고 추가결제용 링크를 발급해서 사용자에게 전달하고 사용자는 결제 완료 확인 & 주문 완료 페이지에서 2번 결제된것 확인가능
>
> 추가적으로 저때 사용자의 결제취소는 직접할 수 없고 CS를 통해서만 가능

## 비즈니스 배경

- 주문 확정 후 추가 비용이 발생하는 경우가 있음 (옵션 변경, 객실 업그레이드, 추가 서비스 등)
- 현재는 이를 처리할 공식적인 수단이 없어 수동 대응 필요
- 추가결제 취소는 복잡도가 높아 CS를 통한 안전한 처리를 우선함
